Welcome to our Web Application Programming and Hacking Course and its overview

Course Overview:

In this course, students will study basic web application development with front-end (HTML5, JavaScript, CSS) and back-end (PHP/MySQL). Web application vulnerabilities and attacks will be introduced and explored with hands-on exercises on the range. Secure programming principles and practices will be introduced to avoid potential web application vulnerabilities and attacks.

LAB-1 :

Lab 1 covers the HTTP protocol, the Web, and the fundamentals of web application development. I became acquainted with the HTTP protocol and the Wireshark program in the first section by looking at HTTP streams, requests, and responses. This required utilizing Wireshark to capture network data, examining headers, and comprehending HTTP communication flow. Furthermore, I sent a small amount of HTTP requests using telnet and used Wireshark to examine the messages that came back, identifying variations between requests made through telnet and those made through a browser. In the second section, I went further into the fundamentals of web application development and made straightforward PHP and CGI web apps. These projects required creating, assembling, and implementing PHP and CGI applications, displaying expertise in both HTTP and server-side programming. I had practical exposure to basic web technologies and tools throughout the lab, which was helpful for my future research into web development and security.

HACKATHON-1:

This hackathon provided me with a full understanding of XSS techniques, including both attacks and defensive strategies. I understand the different attack vectors, such as Stored XSS, Reflected XSS, and DOM-based XSS, and how they can be carried out via input fields, URLs, or HTTP headers. I am skilled in identifying the possible effect of XSS attacks, ranging from session hijacking to website defacement. In addition, I've looked at preventive measures including input validation, output encoding, and the use of security headers like Content Security Policy (CSP). My knowledge extends to both client-side and server side languages, allowing you to traverse XSS vulnerabilities regardless of technological stack. With this understanding, I am well-equipped to efficiently identify and remediate XSS vulnerabilities, assuring security and integrity.
